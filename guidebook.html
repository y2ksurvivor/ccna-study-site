Getting Started — 02

Navigation & Layout

A sidebar on the left, a content area on the right, and a top bar showing where you are.

Sidebar

Click any tool in the left sidebar to switch to it. Tools are grouped under Study, Practice, and Tools. The currently active tool is highlighted.

On mobile, tap the hamburger icon (top-left) to open and close the sidebar.

Top Bar

The top bar shows the current tool name and a metadata chip (e.g. total card count, question count). This updates automatically when you switch tools.

Progress is Automatic

All study progress — correct/incorrect counts per card, quiz scores, and best times — is saved to your browser's localStorage. It persists across page refreshes as long as you use the same browser.

Tip: Clearing your browser's site data will reset all progress. If you want a fresh start on a specific deck, use the "Reset" option inside that tool.
Study — 03

Flackbox Deck

589 concept flashcards drawn directly from the Flackbox CCNA 200-301 Anki deck, covering Sections 3 through 30.

Setup

When you open the Flackbox Deck you land on the setup screen. From here you can:

  • Select a specific section to drill (e.g. Section 21 – VLANs & Trunking) or leave it on All Sections.
  • See your current progress stats — total cards, how many you've answered correctly and incorrectly, and your coverage percentage.
  • Choose between Flashcard Mode or Quiz Mode.
Mode 1
Flashcard Mode

See the question, flip the card to reveal the answer, then mark it as Got It or Review Again. Cards you miss are re-queued at the end of the session.

Mode 2
Quiz Mode

Four multiple-choice options. Select your answer — correct answers highlight green, wrong answers highlight red and reveal the correct option. Explanation auto-opens on wrong answers.

Adaptive Weighting

The deck uses a weighted shuffle — cards you've gotten wrong repeatedly appear more often, cards you've mastered appear less. This means the deck automatically focuses on your weak spots without you having to configure anything.

Keyboard Shortcuts

  • Space or Enter — flip the card (Flashcard Mode)
  • 1 2 3 4 — select an option (Quiz Mode)
  • Enter — advance to the next card after answering
Study — 04

AI Study Deck

AI-generated multiple-choice questions with full explanations, covering 18 exam topics at three difficulty levels.

Setup Filters

Before starting, you can filter by:

  • Topic — OSI, Ports, OSPF, EIGRP, STP, VLANs, Subnetting, NAT/ACLs, BGP, IPv6, Wireless, FHRP, Switch Security, Automation, WAN, and more.
  • Difficulty — Easy, Medium, or Hard.

The setup screen shows how many questions match your current filter, and your historical correct/incorrect ratio for that combination.

How Questions Work

Each question includes:

  • Four answer choices labeled A–D
  • A full explanation (shown automatically when you answer wrong, or expandable when correct)
  • A study tip or mnemonic where applicable
  • A difficulty indicator dot (green = easy, yellow = medium, red = hard)

Results Breakdown

At the end of each session, you'll see a score circle and a per-topic breakdown table with color-coded bars. Topics below 70% are flagged — use that to decide which section to drill next.

Study Strategy: Start with All Topics / Easy. Once you consistently hit 80%+, move to Medium. Use the results breakdown to identify which topics to isolate.
Practice — 05

Command Recall

A typing-speed game where you race to type Cisco IOS commands from memory. Think of it as the keyboard version of muscle memory training.

How It Works

  1. A scenario is shown — e.g. "Save the running config to startup config."
  2. Type the full command from memory.
  3. Each character is checked as you type — green for correct, red for wrong.
  4. Finish the command and move to the next one.

Scoring

Your WPM (words per minute) and accuracy percentage are tracked per session. Aim to get accuracy above 95% before optimizing for speed.

Tip: This tool is best used after you already know the commands — use Commands Puzzle to learn them first, then Command Recall to lock in the muscle memory.
Practice — 06

Subnet Methods

Two dedicated drills for the two most important subnetting calculation methods on the CCNA exam.

When you open Subnet Methods you'll see a menu with two options. Pick one to begin.

Method 1
Block Size

Given a random IP + prefix (/25–/30), you calculate the full subnet profile step by step.

Method 2
Powers of 2

Fill in the full 8-row 2ˢ / 2ʰ−2 table from memory, timed, with your best time saved.

Block Size Drill — Step by Step

Given an IP like 192.168.10.130/26, you answer these six fields in order:

  1. Block size — 256 − mask octet. For /26 the mask octet is 192, so block size = 64.
  2. Network address — the start of the subnet block the IP falls in.
  3. First usable host — network address + 1.
  4. Last usable host — broadcast − 1.
  5. Broadcast address — last address in the block.
  6. Next network — network address + block size.

Each answer is checked immediately. Wrong answers shake the input and let you retry — no attempt limit. Your total completion time is shown at the end.

Powers of 2 Drill

An 8-row table appears with two blank columns:

  • 2ˢ — number of subnets created by borrowing that many bits
  • 2ʰ − 2 — number of usable hosts when that many bits remain for hosts

The timer starts on your first keystroke. Each cell turns green and locks when correct. Hover over any cell to see a formula hint (e.g. 2³ or 2³ − 2). When all 16 cells are filled correctly, your time is recorded and compared to your personal best.

Goal: Get the Powers of 2 table under 30 seconds. At that point it's automatic and you can use it on any exam question without thinking.
Practice — 07

Commands Puzzle

Fill-in-the-blank IOS command construction. A scenario is given, one critical word is blanked out — type the missing word.

How It Works

  1. Select a section (or All Sections) from the category picker.
  2. Read the scenario describing what the command does.
  3. The command is shown with one key word replaced by _______.
  4. Type the missing word and press Enter or click Submit.
  5. You have 5 attempts before the answer is revealed.

Answer Reveal — CLI Sequence

When the puzzle ends (correct or not), a terminal window appears showing the full command sequence an engineer would type on a real device — from User EXEC mode through all navigation steps to the final command. Navigation steps are dimmed; the answer is highlighted in green with a blinking cursor.

For example, a config-if mode command would show:

  • Switch> enable
  • Switch# configure terminal
  • Switch(config)# interface GigabitEthernet0/1
  • Switch(config-if)# switchport mode access ← your answer, highlighted

Skip & Show Answer

If you have no idea, click the Skip & Show Answer link below the input. The answer is revealed immediately and the puzzle is re-queued — it will come back as your very next question so you get an immediate retry.

Keyboard Shortcuts

  • Enter — submit your answer
  • Enter on result screen — go to next puzzle

Hint System

After two wrong attempts, a hint appears automatically showing the first word(s) of the command and the total word count.

Practice — 08

Octet Drill

Binary-to-decimal and decimal-to-binary conversion for individual octets, under time pressure.

The exam doesn't require you to convert full IPs from scratch, but you need to instantly recognize values like 128, 192, 224, 240, 248, 252 — the subnet mask octets. This drill builds that recognition.

How It Works

You're shown an octet value (decimal or binary) and asked to type the other form. A timer tracks each answer. Your streak and average response time are shown as you go.

Focus on: The 8 power-of-2 values (128, 64, 32, 16, 8, 4, 2, 1) and the 6 subnet mask values (128, 192, 224, 240, 248, 252). Those cover 90% of exam scenarios.
Practice — 09

2ⁿ Logic

Quick-fire questions that test whether you can apply powers of 2 to real subnetting problems.

Where the Powers of 2 drill in Subnet Methods trains you to recall the table, 2ⁿ Logic trains you to apply it. Questions take the form of:

  • "A /26 gives how many usable hosts?"
  • "You need at least 50 hosts per subnet — what is the minimum prefix length?"
  • "Borrowing 4 bits from a /24 gives how many subnets?"

Single answer per question, immediate feedback, scored by session.

Practice — 10

Division Method

Subnetting using the division / magic number approach — given a network and a requirement, find the right subnet.

The Division Method is an alternative to Block Size for finding subnet boundaries. Rather than working from 256 − mask octet, you divide the address space by the number of required subnets to find the block size directly.

Questions present a network address and a host/subnet requirement. You calculate and enter the subnet mask, block size, and the specific subnet a host IP belongs to.

Tools — 11

IOS Command Ref

A searchable, filterable reference for every major Cisco IOS command covered on the CCNA exam.

Search

Type anything in the search bar — command name, keyword, concept, or tag (e.g. "OSPF neighbor", "trunk", "show interface"). Results update in real time across command names, syntax, descriptions, variations, output tips, and tags.

Category Filters

Use the filter bar to narrow results by category: Interfaces, Routing, OSPF, EIGRP, STP, VLANs, DHCP, NAT, Security, and more. You can combine search and filter.

Command Cards

Each card shows the command name, syntax, and a short description. Click any card to open the full detail modal, which includes:

  • Full syntax with all forms
  • Complete description
  • Command variations (e.g. show interfaces status vs show interfaces summary)
  • Output tips — what to look for in the output
  • Study tip
  • Searchable tags
Tip: Use this as a reference after getting a question wrong in Flackbox or Commands Puzzle — look up the command to understand the full context, not just the answer.
Tools — 12

Subnet Calculator

Enter any IPv4 address and prefix — get back the full subnet profile instantly.

How to Use

  1. Enter an IP address in dotted-decimal format (e.g. 192.168.10.130).
  2. Enter the prefix length (e.g. /26) or subnet mask.
  3. Results display immediately: network address, broadcast, first/last usable host, block size, total and usable host count, and the next network.

This is a reference tool — use it to verify your manual calculations from the Block Size or Division Method drills. Don't rely on it during study sessions; use it to check your work afterward.

Tools — 13

ACL Builder

A visual tool for constructing and understanding IPv4 ACL statements.

How to Use

The ACL Builder walks you through each component of an ACL statement interactively:

  • ACL type — Standard (number 1–99) or Extended (100–199), or named
  • Action — permit or deny
  • Source / Destination — IP address with wildcard mask
  • Protocol & Port — for extended ACLs (TCP, UDP, ICMP, etc.)

As you build the statement, the full IOS syntax is shown in real time. The explainer below it describes in plain English what the statement will do.

Wildcard Mask Reminder: Wildcard masks are the inverse of subnet masks. A /24 subnet mask of 255.255.255.0 has a wildcard of 0.0.0.255. The ACL Builder will calculate this for you if you enter the prefix.
